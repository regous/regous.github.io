修改swashbuckle，升级swaggerui

swashbuckle是测试.Net WebAPI的库，基本的功能都有了，界面也比较美观。最近在做一个项目的时候，用到了上传文件的接口，swashbuckle-5.3.1不能支持多个文件，而OpenAPI3.0是支持的，swagger-ui，在前几天（10月13日）更新到了3.3.2，界面更加美观了，作为一个好色之徒，这个理由足够让我升级了。

前置工作

建一个WebAPI工程，在从nuget中添加swashbuckle引用，这时引用的是5.3.1版本的，参考 如何使 WebAPI 自动生成漂亮又实用在线API文档 ，配置工程，配置完成后，应该就能满足大部分的需求了。

不折腾会死

在github上clone代码 swagger-ui 和 swashbuckle 。

CustomAsset函数第一个参数相当于键，同时也是作为URL的一部分，因此不能带'.'，否则IIS会以文件的方式进行处理，但是这些文件，我们是以嵌入的资源进行处理的。

修改index.html

修改参数的URL路径，由：

url: "http://petstore.swagger.io/v2/swagger.json"

改为：

url: '%(Url)'

由：

<link rel="stylesheet" type="text/css" href="./swagger-ui.css" >
<link rel="icon" type="image/png" href="./favicon-32x32.png" sizes="32x32" />
<link rel="icon" type="image/png" href="./favicon-16x16.png" sizes="16x16" />
<script src="./swagger-ui-bundle.js"> </script>
<script src="./swagger-ui-standalone-preset.js"> </script>

改为：

<link href='css/swagger-ui-css' rel='stylesheet' type='text/css' />
<link rel="icon" type="image/png" href="img/favicon-16x16-png" sizes="32x32" />
<link rel="icon" type="image/png" href="img/favicon-32x32-png" sizes="16x16" />
<script src="js/swagger-ui-bundle-js"> </script>
<script src="js/swagger-ui-standalone-preset-js"> </script>

如果无法访问，则通过你懂的，下载到本地再添加到工程的资源里。

js/css/png的href与CustomAsset函数的第一个参数相同。

设置%(Url)

在EmbeddedAssetProvider.GetEmbeddedResourceStreamFor里

private Stream GetEmbeddedResourceStreamFor(EmbeddedAssetDescriptor resourceDescriptor, string rootUrl)
{
    var stream = resourceDescriptor.Assembly.GetManifestResourceStream(resourceDescriptor.Name);
    if (stream == null)
        throw new AssetNotFound(String.Format("Embedded resource not found - {0}", resourceDescriptor.Name));

    if (resourceDescriptor.IsTemplate)
    {
        var templateParams = _templateParams
            .Union(new[] { 
                new KeyValuePair<string, string>("%(RootUrl)", rootUrl), 
                new KeyValuePair<string, string>("%(Url)", string.Format("{0}/{1}", rootUrl, _templateParams["%(DiscoveryPaths)"].Split('|')[0])) 
            })
            .ToDictionary(entry => entry.Key, entry => entry.Value);

        return stream.FindAndReplace(templateParams);
    }

    return stream;
}

改完编译通过后，移除原来对Swashbuckle.Core的引用，再引用自己的编译的Swashbuckle.Core.dll就可以了。
